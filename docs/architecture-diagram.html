Key Request Flows

Flow 1 Authentication

1
User → Frontend› Submit credentials
2
Frontend → Backend /auth› POST login
3
Backend → Snowflake› SHOW GRANTS (fetch roles)
4
Redis› Cache role set (TTL)
5
Backend → Frontend› JWT + role payload
6
Frontend› UI adapts dynamically to roles

Flow 2 AI Model Execution

1
User → Frontend› Trigger skill execution
2
Backend /execute› Auth + JWT check
3
Permission Service› Validate Snowflake role
4
Token Service› Quota check + cost estimate
5
MCP Bridge → AI Model› Execute with session
6
Response → Backend› Tokens counted
7
DB + Snowflake› Write usage + audit log

Flow 3 Access Request

1
User → Request Access Page› Submit form
2
Backend /requests› Write to ACCESS_REQUESTS
3
Admin → Admin Panel› Review request
4
Admin approves› Backend processes approval
5
Snowflake› Update AI_MODEL_ACCESS_CONTROL
6
Redis› Invalidate role cache
7
RBAC_AUDIT_LOG› Write governance record
